# TSK0001 — OTP cleanup and auth logging

## Status
Implemented

## Context
During deployment testing, OTP email delivery worked, but verification sometimes returned "invalid or expired OTP" after a failed attempt. Backend logs did not show enough information to diagnose auth failures.

## Problem
1. OTP state may not be cleaned or replaced correctly.
2. Auth/OTP flow has insufficient backend logging.

## Scope
- Fix OTP replacement/cleanup behavior.
- Add safe structured logs for OTP/auth events.
- Add focused tests.

## Out of Scope
- Redis
- Database-backed OTP storage
- Full Spring Security migration
- Full audit subsystem
- Complex rate limiting

## Acceptance Criteria
- Requesting a new OTP replaces any previous pending OTP for the same email/mobile.
- Wrong OTP does not poison the challenge.
- Correct OTP works after a previous wrong attempt.
- Expired OTP fails clearly and is cleaned up.
- Successful verification clears pending OTP state.
- Logs exist for request, send, verify success, and verify failure reason.
- OTP code is never logged.
- Secrets are never logged.
- Tests pass.

## Notes
Keep the implementation small and local to the existing auth flow.
