# TSK0005 — Normalize portal Markdown documents

## Status
In progress

## Context
Some project Markdown documents are too complex for the structured Markdown editor.

The portal supports a controlled Markdown subset. Existing documents should be simplified so they render, edit, save, and reload predictably.

## Scope
- Review existing portal Markdown documents.
- Simplify documents to the supported Markdown subset.
- Use only `#`, `##`, and `###` headings.
- Keep editable content as plain paragraphs.
- Keep technical readonly content in fenced code blocks.
- Normalize simple Markdown tables.
- Escape `|` inside table cells as `\|`.
- Avoid inline triple backticks and complex Markdown constructs.
- Add one canonical example document for future reference.

## Out of Scope
- Editor code changes.
- Full Markdown support.
- AI rewriting.
- Backend or storage changes.
- Automatic migration scripts.

## Acceptance Criteria
- Documents render correctly in the portal preview.
- Paragraphs are editable as textareas.
- Table cells are editable as inputs.
- Code blocks and unsupported content remain readonly.
- Save and reload do not corrupt document structure.
- Edit footer is appended correctly.
- Markdown authoring rules are documented briefly.

## Risks
- Complex Markdown may still render as readonly/raw.
- Tables may break if edited manually outside the portal.
- Scope may drift into building a full Markdown editor.
