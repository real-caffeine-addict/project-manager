# TSK0003 — Structured Markdown form editor

## Status
In progress

## Context
The current Markdown editor is too raw for non-technical users, especially around headings, fenced code blocks, and tables.

The goal is to let users edit document text through structured React UI elements without directly editing Markdown syntax.

## Scope
- Parse supported Markdown into a frontend block model.
- Render `#`, `##`, `###` as readonly headings.
- Render paragraphs as editable textareas.
- Render fenced code blocks as styled readonly code blocks.
- Render simple Markdown tables as HTML tables.
- Render table cells as editable inputs while keeping table structure fixed.
- Serialize the block model back to Markdown on save.
- Save through the existing backend endpoint.
- Append edit footer with current user/email and timestamp.

## Out of Scope
- Full Markdown editor
- `contenteditable`
- `dangerouslySetInnerHTML`
- Raw HTML rendering
- Editing code block content
- Adding/removing table rows or columns
- Database/storage changes
- AI assistance

## Rules
- Preserve table row/column count.
- Escape `|` inside table cells as `\|`.
- Unsupported Markdown should be preserved or shown readonly.
- Frontend sanitation is for usability only; backend validation remains required.

## Acceptance Criteria
- Headings render visually.
- Paragraph text can be edited and saved.
- Code blocks render styled and remain unchanged after save.
- Tables render as HTML tables.
- Table cell values can be edited without changing table structure.
- Saved Markdown remains valid.
- Save appends an edit footer.
- Existing auth and suggestion/comment functionality still works.
- Frontend build passes.

## Risks
- Markdown round-trip may corrupt formatting.
- Table parsing can break edge cases.
- Scope can expand into a full Markdown editor if not controlled.
